### Adding New Documentation

**User Guide**: Place in `docs/guides/`
```bash
# Example

```

**Admin Documentation**: Place in `docs/admin/`
```bash

```

**Deployment Guide**: Place in `docs/deployment/`
```bash

```

**Implementation Notes**: Place in `docs/implementation/`
```bash
# Example
docs/implementation/NEW_API_INTEGRATION.md
```

### Documentation Naming Convention
- Use UPPERCASE for main words
- Separate with underscores
- Include type suffix: `_GUIDE`, `_REFERENCE`, `_SETUP`, `_COMPLETE`
- Examples: `DATABASE_SETUP.md`, `ADMIN_QUICK_REFERENCE.md`
